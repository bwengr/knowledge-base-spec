# Contributing

Contributions are welcome! This spec aims to be a community-driven standard for structured knowledge bases.

## How to Contribute

### Proposing Changes

1. Fork the repository
2. Create a branch for your change
3. Make your changes
4. Submit a pull request with a clear description of why the change improves the spec

### Types of Contributions

- **Spec improvements** - Clarify existing sections, add missing use cases
- **New section types** - Propose additional content types with justification
- **Examples** - Add real-world examples from different industries
- **Tools** - Add generators, validators, or integrations in the tools directory
- **Documentation** - Improve clarity, add translations

### Standards for Changes

- Changes must maintain backward compatibility when possible
- New features need clear rationale and example usage
- All JSON examples must validate against the schema

## Getting Help

Open an issue if you have questions or want to discuss a change before implementing.
